# v1.23 — исправление ложной успешной сборки и заглушек в C++ коде

Изменения:

1. Исправлена команда прямой сборки C++ на Windows.
   Раньше команда начиналась с:

   ```bat
   if not exist build mkdir build && g++.exe ...
   ```

   Если папка `build` уже существовала, `cmd.exe` мог пропустить оставшуюся часть строки, вернуть `PROCESS_EXIT_CODE: 0`, но не создать `build/app.exe`.

   Теперь используется безопасная схема:

   ```bat
   if not exist build mkdir build & if exist build\app.exe del /q build\app.exe & g++.exe ...
   ```

   Старый exe удаляется перед сборкой, поэтому проверка артефакта не может принять старый файл за новый результат.

2. `run_command` автоматически нормализует старые команды вида:

   ```bat
   if not exist build mkdir build && ...
   ```

   в безопасный вариант.

3. Если в исходниках C++ обнаружены признаки заглушек или неполной реализации (`conceptual`, `omitted`, `TODO`, `заглушка`, `не реализован` и т.п.), задача не считается завершённой.

4. В системный prompt добавлено правило: нельзя объявлять C++ задачу завершённой при `BUILD_ARTIFACT_MISSING`, ненулевом `EXIT_CODE` или заглушках в исходниках.

5. Исправлена подсказка для Windows-путей: в prompt теперь показывается `src\\main.cpp` и `build\\app.exe`, а не склеенное `srcmain.cpp`/`buildapp.exe`.
