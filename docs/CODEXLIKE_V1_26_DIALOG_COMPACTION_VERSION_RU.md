# v1.26 — объединение повторяющихся сообщений и версия внизу интерфейса

## Что исправлено

1. Повторяющиеся сообщения агента о цикле компиляции, `exit code 1/2`, `BUILD_ARTIFACT_MISSING`, `run_command` и CMD больше не должны засорять диалог отдельными одинаковыми карточками.
2. Старые уже сохранённые диалоги отображаются компактно: повторяющиеся ответы сворачиваются в одну карточку со скрытым блоком **«Попытка компиляции»**.
3. Повторяющиеся ошибки подключения к модели, например `SocketException`, `HTTP 400`, `Failed to load model`, сворачиваются в скрытый блок **«Ошибки подключения к модели»**.
4. Новые повторяющиеся сообщения во время выполнения задачи записываются в общий action-блок текущего сообщения прогресса, а не добавляются каждый раз в чат отдельной видимой карточкой.
5. Если агент остановился после нескольких попыток, итоговое сообщение сохраняет скрытые блоки с попытками, чтобы можно было открыть подробности без мусора в диалоге.
6. Внизу интерфейса добавлена строка версии программы: `v1.26`.

## Зачем это сделано

В логах и сохранённом диалоге было видно, что модель многократно повторяла почти одинаковый текст о том, что компиляция не работает из-за `run_command`, `exit code 2` и `BUILD_ARTIFACT_MISSING`. Такие ответы не давали новой информации и раздували историю. Теперь они группируются и не мешают работе.
