# v1.22 — исправление прямой C++ сборки MinGW/MSYS2 и поведения после логов

Исправлено по логам `Вставленный текст(18).txt`:

- `g++.exe "src/main.cpp"` под Windows/MSYS2 мог передавать кавычки дальше в линкер, из-за чего появлялась ошибка `cannot find "src/main.cpp": Invalid argument`.
- Прямая C++ сборка теперь формируется как `g++.exe -std=c++17 -O2 src\\main.cpp -o build\\app.exe` без кавычек для простых относительных путей.
- `run_command` дополнительно нормализует пользовательские/модельные команды: кавычки вокруг простых `*.cpp/*.cc/*.cxx` путей убираются автоматически.
- После сборки по-прежнему проверяется фактический артефакт `build/app.exe`; если файла нет, возвращается `BUILD_ARTIFACT_MISSING` и задача не считается успешной.
- В память решений больше не подмешивается бесполезная команда `cd`, повторяющиеся решения дедуплицируются, а старые команды с кавычками вокруг `src/main.cpp` в подсказках показываются уже в безопасном виде.
- Устранена случайная дублированная строка объявления `quickPanel()` во вкладке консоли.

Проверка архива: `unzip -t`.
Flutter/Dart SDK в среде сборки отсутствует, поэтому `flutter analyze` и Windows-сборка не выполнялись.
