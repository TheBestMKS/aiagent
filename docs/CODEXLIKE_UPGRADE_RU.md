# Усиление ИИ-агента до Codex/OpenCode-like режима

В этой версии усилено ядро агента, чтобы он не останавливался после обычного текстового плана и был ближе к полноценному Codex/OpenCode-подобному исполнителю задач.

## Что изменено

1. **Цикл агента**
   - Агент больше не считает текстовый план выполнением задачи.
   - Если задача требует файлов, сборки или запуска, но модель не вызвала инструменты, программа отправляет модели принудительное продолжение.
   - Исходная задача пользователя хранится отдельно в `activeTaskText` и всегда добавляется в системный контекст, чтобы не потеряться при сжатии истории.
   - Ведётся состояние задачи: количество действий, изменений файлов, запусков команд, неудачных команд и последний exit code.

2. **Проверка качества**
   - Если включена проверка качества, агент требует:
     - создать/изменить файлы для задач разработки;
     - выполнить `run_command` или `run_tests` после изменений;
     - исправлять ошибки, если последняя команда завершилась с ненулевым exit code.

3. **Права и разрешения**
   - Режимы прав теперь реально применяются перед выполнением инструментов.
   - В режиме запроса разрешений появляется диалог подтверждения.
   - В режиме критичных действий запрашиваются запуск команд, удаление, перемещение, копирование и распаковка.
   - Чтение файлов и просмотр проекта разрешаются автоматически.

4. **Постоянные логи**
   - Помимо окна логов, теперь пишутся файлы:
     - `.cppagent/logs/latest.log`
     - `.cppagent/logs/<timestamp>.log`
     - `.cppagent/logs/actions.jsonl`
   - В `actions.jsonl` пишутся структурированные события: запрос пользователя, tool start/finish/error, запись файлов, команды, разрешения, запуск llama.cpp.

5. **Диагностика llama.cpp**
   - Приоритет поиска backend изменён на `CUDA → Vulkan → CPU`.
   - Логируются exe, рабочая папка, backend, модель, аргументы запуска.
   - Читаются stdout/stderr процесса `llama-server`.
   - Выполняется healthcheck `/v1/models`; статус “запущен” ставится только после успешного ответа API.

6. **Контекст и max_tokens**
   - `max_tokens` больше не жёстко равен 4096.
   - Перед запросом оценивается размер prompt и вычисляется безопасный output budget.
   - История tool-result сжимается, а исходная задача сохраняется отдельно.

7. **Файловая безопасность**
   - Инструменты проекта теперь запрещают абсолютные пути вне текущего проекта.
   - Для внешних вложений используется отдельная логика `resolvePossiblyRelativePath`.

8. **run_tests по умолчанию**
   - Добавлены более полезные команды для Flutter, CMake, npm, Cargo, pytest, `main.py`, `main.cpp`.

## Важное ограничение

Это всё ещё локальный агент поверх текстовых tool-call. Его качество зависит от модели. Но теперь программа не доверяет модели “на слово” и принудительно продолжает задачу, если нет файлов, проверки или исправления ошибок.
