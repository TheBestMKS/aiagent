# AI Agent v1.47 — контроль проекта, повторные правки, почта и трей

## Что исправлено

- Добавлен preflight-аудит проекта перед запуском задачи: агент получает сведения о типе задачи, корневых файлах и правилах проверки соответствия проекта запросу пользователя.
- Для задач разработки ПО автоматически резервируются явные старые артефакты сборки (`CMakeFiles`, `CMakeCache.txt`, `build.ninja`, `cmake_install.cmake`, `Makefile`) в `.cppagent/reserved_before_task/<timestamp>/`, чтобы CMake не цеплялся за следы прошлых задач.
- Исправлен парсер XML `<tool_call>`: теперь он берёт весь блок между тегами, а не первую пару `{}`. Это важно для `write_file`, потому что код C++/Dart/PHP содержит фигурные скобки.
- Добавлен ограничитель повторных изменений одного файла: если агент много раз подряд изменяет один и тот же файл и не запускает проверку/сборку, действие блокируется и модели возвращается требование перейти к `run_tests`, `run_command`, `project_map` или `read_file`.
- Усилен системный промт по типам задач: разработка ПО, работа с файлами устройства, документы, презентации, администрирование, почта.
- Добавлены настройки почтовых аккаунтов: адрес, IMAP, SMTP, логин, пароль приложения, SSL/TLS.
- Добавлены инструменты агента `email_list_accounts` и `email_draft_smtp` для работы с почтой. Отправка письма без подтверждения пользователя не выполняется.
- Добавлены настройки Windows-трея: сворачивание в трей при закрытии и уведомления о ходе выполнения. Dart вызывает нативный Windows bridge, если он доступен в собранной версии.

## Причина зависания на Matrix.h

Локальная модель вернула `<tool_call>` как обычный текст с большим JSON, внутри которого был C++ код с `{}`. Старый парсер пытался вытащить JSON регуляркой до первой `}`, поэтому инструмент `write_file` фактически не выполнялся как надо, и цикл повторял попытки. В v1.47 парсер исправлен.
