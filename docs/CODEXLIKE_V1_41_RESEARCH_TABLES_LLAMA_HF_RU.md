# AI Agent v1.42 — поиск, таблицы, документы, llama.cpp и HuggingFace

## Исправления поведения агента

- Для чистых задач поиска информации агент больше не должен запускать `run_command` или `run_tests` после `web_research`.
- После успешного `web_research` цикл завершается итогом по источникам.
- Для поисков людей источники по другим людям не должны попадать в основной список подтверждённых источников.

## Диалоги и вывод

- Постоянная нижняя панель с версией и статусом удалена.
- Markdown-таблицы в сообщениях теперь отображаются как таблицы, а не как простой текст.
- Найденный текст файлов из папки выводится скрытыми раскрываемыми блоками кода.

## Извлечение текста из файлов

- Для PDF добавлен безопасный режим: если текст нельзя корректно извлечь, программа сообщает об этом и не выводит каракули.
- Для старых бинарных Office-форматов `.doc/.xls/.ppt` добавлена фильтрация мусора и диагностическое сообщение вместо кракозябр.
- Для `.docx/.xlsx/.pptx` сохраняется извлечение через внутренние XML-файлы.
- Для PDF при наличии `pdftotext` в `tools` используется он.

## llama.cpp

- В настройках моделей добавлена загрузка llama.cpp из последнего релиза GitHub `ggml-org/llama.cpp`.
- Можно выбрать режим: CPU, Vulkan, CUDA.
- Скачивание выполняется в `tooling/downloads`, распаковка — в `tooling/llama.cpp/<режим>`.
- При запуске локального llama.cpp отображается маленькая строка с занятым ОЗУ.
- Профили llama.cpp поддерживают `mmproj-*.gguf` через аргумент `--mmproj`.

## HuggingFace

- Добавлен поиск моделей на `https://huggingface.co/` через API.
- Можно выбрать репозиторий, увидеть `.gguf` и `mmproj-*.gguf` файлы и скачать выбранный файл в `models/<repo>/...`.

## Релизы

- Скрипты сборки обновлены до v1.42.
- Копирование Windows и Android артефактов в `dist` стало более устойчивым: если стандартный путь не найден, скрипт ищет релизный exe/APK в `build`.
