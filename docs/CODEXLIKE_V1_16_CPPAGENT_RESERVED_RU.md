# v1.16 — `.cppagent` только для служебных данных

Исправлена проблема, когда модель могла создавать пользовательские исходники и `CMakeLists.txt` в служебной папке `.cppagent`.

## Новые правила

- `.cppagent` предназначена только для внутренних данных агента: логи, планы задач, сессии, временные файлы, память решений и `.cppagent/python_venv`.
- `write_file`, `append_file`, `replace_text`, `make_dir`, `delete_path`, `copy_path`, `move_path` блокируют создание/изменение пользовательских файлов внутри `.cppagent`.
- `project_map` и `list_files` скрывают `.cppagent`, чтобы модель не воспринимала её как рабочую папку проекта.
- В системный prompt добавлено прямое правило: исходники писать в корень проекта или `src/`, `include/`, `tests/`, `data/`, `assets/`, `build_scripts/`.
- Если старые версии агента уже создали рабочие файлы в `.cppagent`, модель должна скопировать полезное содержимое в нормальную папку проекта и продолжить там.
