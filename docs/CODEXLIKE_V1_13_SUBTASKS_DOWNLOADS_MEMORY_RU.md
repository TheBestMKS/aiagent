# v1.13 — подзадачи, загрузка ПО в tools, память решений и полный диалог

## Что добавлено

1. Агент получил инструмент `set_task_plan` для фиксации плана сложной задачи в `.cppagent/task_plan.md`.
2. В системные инструкции добавлено обязательное разбиение сложных задач на этапы: анализ среды, план, создание/изменение файлов, проверка, исправление ошибок, повторная проверка, итог.
3. Добавлены инструменты:
   - `download_to_tools` — скачать файл или архив в папку `tools`;
   - `extract_zip_to_tools` — распаковать ZIP в `tools`, например `tools/windows/x64`;
   - `remember_solution` — сохранить найденный способ решения ошибки в память агента.
4. Агент передаёт модели компактную память решений из `config/solution_memory.jsonl`.
5. При успешном выполнении команды после предыдущих ошибок агент автоматически сохраняет рабочую команду как подсказку для будущих задач.
6. При ошибках выполнения агент теперь просит модель продолжать решение как набор подзадач: проверить тип ошибки, проверить `tools`, выбрать исправление, скачать/распаковать ПО при необходимости, повторить команду.
7. Диалог в UI больше не обрезается по `visibleExchangeCount`: показывается вся публичная история сессии.
8. Когда контекст для модели автоматически сжимается/часть старой истории не помещается, в диалоге добавляется разделитель `Автоматическое сжатие контекста`.

## Важные правила

- Скачанное ПО по умолчанию устанавливается/распаковывается только в `tools`, а не в системные папки.
- На Windows x64 приоритетная папка: `tools/windows/x64`.
- Python-пакеты по-прежнему ставятся только в проектное окружение `.cppagent/python_venv`.
- Если отсутствует компилятор или утилита, это ошибка окружения. Агент не должен ухудшать исходный код, а должен искать инструмент в `tools`, скачать/настроить его или сообщить пользователю после исчерпания вариантов.
