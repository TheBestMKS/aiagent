# v1.24 — Интернет-инструменты и принудительная самописная реализация

## Что исправлено

1. Добавлены инструменты для работы с интернетом:
   - `duckduckgo_search` — поиск через DuckDuckGo HTML/API-совместимый поиск;
   - `web_fetch` — открытие страницы и возврат очищенного текста/ссылок;
   - `download_to_project` — скачивание файла внутрь текущего проекта.

2. Уточнены правила поведения агента:
   - если пользователь запретил дополнительные библиотеки или просит писать код самостоятельно, агент обязан писать самописную реализацию;
   - если пользователь не запретил зависимости и задача реально требует тяжёлую библиотеку/SDK, агент может искать официальный источник, скачивать и подключать зависимость;
   - если пользователь просит найти реализацию в интернете и установить её в проект, агент использует поиск, открытие страниц, скачивание и адаптацию файлов проекта;
   - агент не должен завершать задачу словами «нужны тяжёлые библиотеки» — после такого вывода он должен либо установить/подключить нужное, либо написать рабочую минимальную реализацию сам.

3. Исправлена обработка испорченных tool-call от локальных моделей:
   - теперь распознаётся формат вида `<tool_call>write_file{content:<|"|>...<|"|>,path:<|"|>...<|"|>}<tool_call|>`;
   - фиктивные значения `cmake_file_content`, `main_cpp_content`, `code_here`, `placeholder` не записываются в файлы;
   - если модель передала такой placeholder, агент возвращает `PLACEHOLDER_CONTENT_BLOCKED` и требует настоящий полный текст файла.

4. Усилен повторный prompt:
   - модель получает прямое требование не писать один план;
   - для самописных задач требуется писать код по частям, запускать тесты/сборку, исправлять ошибки и продолжать до завершения;
   - для интернет-задач требуется использовать `duckduckgo_search`, `web_fetch`, `download_to_project` или `download_to_tools`.

## Ограничения

- Интернет-инструменты зависят от доступности сети на машине пользователя.
- `duckduckgo_search` использует HTML-выдачу DuckDuckGo без API-ключа, поэтому формат выдачи может измениться.
- `web_fetch` возвращает очищенный текст страницы, а не полноценный браузерный DOM.
- ZIP проверен на целостность, но `flutter analyze` в этой среде не запускался, потому что Flutter/Dart SDK недоступен.
