## Python-политика агента

Агент больше не должен устанавливать пакеты в общий Python пользователя.

Для Python-команд действует правило:

- `pip install ...`
- `pip3 install ...`
- `python -m pip install ...`
- `pytest`
- `python script.py`

будут выполняться через проектное окружение:

```text
<project>/.cppagent/python_venv
```

Если окружения нет, агент создаёт его командой:

```text
python -m venv .cppagent/python_venv
```

Если есть `requirements.txt`, зависимости устанавливаются в это окружение.

Если Python-команда завершилась ошибкой `ModuleNotFoundError: No module named ...`, агент:

1. определяет недостающий модуль;
2. создаёт `.cppagent/python_venv`, если его ещё нет;
3. устанавливает пакет в это окружение;
4. повторяет исходную команду один раз;
5. отдаёт модели stdout/stderr установки и повторного запуска.

## Локальные tools

Папка `tools` остаётся приоритетной. Если пользователь положил Python в:

```text
tools/windows/x64/python.exe
```

или компиляторы/утилиты в подпапки `tools`, агент добавляет их в PATH перед системным PATH.
