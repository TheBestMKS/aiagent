# Исправления v1.7: полноценнее Codex/OpenCode-поведение

## Что исправлено после лога с `finish_reason: length`

1. Внутренние сообщения агента больше не показываются в диалоге как сообщения пользователя.
   - Повторные подсказки модели, tool-result и continuation-сообщения теперь имеют `internal: true`.
   - Они остаются в session.jsonl и в контексте модели, но скрыты из UI и из копирования видимого диалога.

2. Обрезанные ответы модели больше не отображаются пользователю как нормальный ответ.
   - Если модель вернула `finish_reason: length`, не закрыла `<tool_call>` и не дала валидных действий, ответ скрывается из чата.
   - Это предотвращает отображение мусора вроде `Я напи` или незакрытого JSON.

3. Добавлен native OpenAI tools режим.
   - В запрос к OpenAI-compatible API теперь передаются `tools` и `tool_choice: auto`.
   - Если сервер/модель вернёт native `tool_calls`, агент конвертирует их в общий внутренний формат и выполнит.

4. Добавлен локальный fallback для простых C++ задач.
   - Если модель не смогла выдать пригодный tool-call, а задача явно просит C++ программу, агент сам создаёт `main.cpp` и запускает `run_tests`.
   - Это защищает от ситуации, когда маленькая локальная модель несколько раз пишет только reasoning или обрывает tool-call.

5. Добавлен живой прогресс в диалог.
   - Пользовательское сообщение появляется сразу после отправки.
   - На месте будущего ответа появляется статус: запрос модели, запись файла, выполнение инструмента, проверка.

6. Добавлены Codex-like блоки изменений файлов.
   - После записи/изменения/удаления файла в ответе показывается строка файла: `имя +N / -M`.
   - По нажатию блок diff раскрывается/скрывается.
   - При открытии другого файла предыдущий diff закрывается.
   - При отправке нового промта раскрытые diff-блоки сбрасываются.

7. Tool-result больше не засоряет видимый чат.
   - Подробности остаются в persistent-логах и внутренней истории.

## Важное ограничение

Flutter/Dart SDK не установлен в среде сборки ChatGPT, поэтому `flutter analyze` и `flutter build windows` здесь не запускались. Проверить нужно локально:

```bat
analyze.bat
run_tests.bat
build_windows.bat
```
