# v1.25 — исправление вечной загрузки Android

## Проблема
В предыдущей версии `HomeScreen` показывался только после завершения `controller.initialize()`.
Если на Android во время инициализации зависало или падало создание папок, чтение настроек, сканирование проектов/моделей или HTTP-проверка профиля, пользователь видел только бесконечный индикатор загрузки.

## Исправлено
- Интерфейс приложения теперь показывается сразу.
- Инициализация запускается после первого кадра интерфейса.
- Ошибка инициализации отображается внутри уже открытого окна, а не оставляет вечную загрузку.
- Добавлена кнопка «Повторить» при ошибке запуска.
- Для Android используется безопасная внутренняя папка данных на базе `Directory.systemTemp/ii_agent_data`, чтобы приложение не пыталось писать в недоступный `Directory.current`.
- Автопоиск локальных моделей/llama.cpp при старте Android отключён, чтобы не блокировать окно.
- Автопроверка OpenAI-compatible моделей при старте Android отключена; пользователь может обновить модели вручную из интерфейса.
- Создание/чтение папки Projects ограничено таймаутами и при ошибке откатывается во внутреннюю папку приложения.
- Переключение проекта остаётся асинхронным с индикатором загрузки проекта.

## Проверка
ZIP проверен на целостность. Flutter SDK в среде генерации отсутствовал, поэтому `flutter analyze` и Android build здесь не запускались.
