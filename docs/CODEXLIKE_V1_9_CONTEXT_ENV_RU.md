# v1.9 — Исправление контекста сервера и ошибок окружения

## Что исправлено

### 1. Контекст модели и фактический `n_ctx` сервера

Агент больше не считает, что ошибка вида `n_keep >= n_ctx` означает, что модель не поддерживает большой контекст. Теперь он различает:

- `profile context window` — ожидаемый контекст модели в профиле агента, например `131072`;
- `server runtime n_ctx` — фактический контекст, с которым текущий экземпляр модели загружен сервером.

Агент опрашивает сервер через дополнительные endpoints:

- `/props`;
- `/v1/props`;
- `/v1/models`;
- `/api/v0/models`.

Если сервер возвращает ошибку `n_keep >= n_ctx`, агент извлекает `n_ctx` из текста ошибки, пишет это в лог и показывает понятное сообщение: модель может поддерживать 131072, но текущий серверный экземпляр загружен, например, с `n_ctx=768`.

Для llama.cpp нужно запускать модель с нужным контекстом, например:

```bat
llama-server.exe --model model.gguf --ctx-size 131072
```

Для LM Studio / OpenAI-compatible UI нужно выставить `Context Length / Context Window = 131072` в настройках загрузки модели.

### 2. Удалён встроенный fallback-код

Агент больше не содержит заготовленную C++ программу и не пишет её вместо модели. Если модель не смогла выдать tool-call, агент просит модель повторить действие через tools. Код пользовательской задачи должна создавать сама модель.

### 3. Ошибка компилятора больше не считается ошибкой кода

Если `stdout/stderr` показывает, что не найден `g++`, `cl`, `clang++`, `cmake` или другая команда, агент считает это ошибкой окружения, а не ошибкой исходного кода.

Теперь агент:

- пишет полный вывод команды в `.cppagent/logs/commands/..._command.log`;
- передаёт stdout/stderr обратно модели;
- декодирует вывод команд через системную кодировку Windows, чтобы не было кракозябр вместо сообщения `не является внутренней или внешней командой`;
- при ошибке `g++ не найден` пробует автоматический fallback через стандартный `run_tests`, где предусмотрены альтернативы `cl.exe` и `clang++`;
- блокирует перезапись существующего исходника после ошибки окружения, чтобы модель не заменила нормальный код на заглушку только потому, что компилятор не установлен.

### 4. Защита от ухудшения кода после ошибки окружения

Если последняя ошибка связана с отсутствием компилятора/инструмента, `write_file` не даст перезаписать существующий файл. Вместо этого вернёт `WRITE BLOCKED` и объяснит модели, что нужно попробовать другой компилятор или сообщить пользователю об установке инструмента.
