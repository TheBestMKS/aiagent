# v1.14 — восстановление C++/CMake сборки и подпапки проекта

Что добавлено:

- `run_command` получил необязательный аргумент `cwd`, чтобы модель могла запускать команды в подпапке проекта.
- `run_tests` теперь ищет C++ исходники в корне и подпапках проекта.
- Для C++ задач агент может создать или исправить минимальный `CMakeLists.txt` только для сборки уже созданного моделью `.cpp` файла. Это не является встроенным решением предметной задачи, код нейросети по-прежнему пишет модель.
- Если CMake сообщает `does not appear to contain CMakeLists.txt`, `Cannot find source file`, `No SOURCES given to target`, агент распознаёт это как ошибку структуры сборки, а не как ошибку логики нейросети.
- Если модель после нескольких подсказок не выполняет действий, агент делает ограниченную автоматическую попытку восстановления через `run_tests` и возвращает результат модели.
- `run_tests` при наличии CMake использует CMake/Visual Studio как рабочий путь сборки на Windows, что помогает, когда `g++` отсутствует, но Visual Studio Build Tools доступны.
