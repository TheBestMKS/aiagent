# V1.18 — компактный tools prompt и корректная обработка fallback-сборки

Исправления по логу `Вставленный текст(15).txt`.

## Что было не так

1. В system prompt попадал слишком большой список программ из `tools/windows/x64/msys64/mingw64/bin`. Из-за этого контекст раздувался ещё до начала реальной работы.
2. Если первая CMake-команда падала, а автоматический fallback через `g++.exe` успешно собирал и запускал проект, tool-result всё равно начинался с исходной ошибки CMake. Модель воспринимала весь результат как провал и снова запускала CMake.
3. Некоторые локальные модели выводили испорченный XML tool-call вида `run_command{command:<|"|>...}`, который агент не распознавал.
4. После ошибки компиляции модель иногда писала видимый текст `Изменён файл: src/main.cpp`, но не вызывала `write_file` или `replace_text`; агент много раз повторял подсказку.

## Что изменено

- В system prompt теперь попадает только компактная выжимка tools: основные компиляторы, build tools, Python/runtime и строка `... ещё N программ`.
- Полный список по-прежнему доступен через `list_local_tools`.
- `localToolsCompactSummary()` сначала сканирует tools широко, а потом выводит только выбранное количество записей, чтобы не потерять `g++.exe`, `python.exe`, `cmake.exe` из-за маленького `maxItems`.
- Если автоматический fallback-сборщик успешно завершился, tool-result получает явный заголовок:

```text
FINAL_STATUS: SUCCESS
[AUTO_BUILD_FALLBACK_SUCCESS]
...
[/AUTO_BUILD_FALLBACK_SUCCESS]
```

- После такого результата модель получает правило: не запускать повторно неудачную CMake-команду, а дать итог.
- Добавлен парсер legacy tool-call для испорченного формата `<tool_call>run_command{command:<|"|>...}`.
- Retry prompt теперь прямо говорит: фраза `Изменён файл` без вызова `write_file/replace_text` не считается действием.
- Автовосстановление, если завершилось exit code 0, может завершить задачу без ещё одного лишнего круга модели.

## Проверка

В этой среде Flutter/Dart SDK отсутствует, поэтому `flutter analyze` и Windows build не запускались. ZIP проверен через `unzip -t`.
