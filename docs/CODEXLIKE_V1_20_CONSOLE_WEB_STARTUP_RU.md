# V1.20 — консоль, Web, быстрый запуск проекта и устойчивый streaming

Изменения относительно V1.19:

- Вкладка «Консоль»: строка ввода команд перенесена вниз.
- Для каждой вкладки консоли хранится история введённых команд.
- ПКМ по строке команды показывает последние команды и найденные исполняемые файлы из `tools`; выбранный пункт вставляется в текущую позицию курсора, а не заменяет всю строку.
- Команды Windows нормализуются: `ls` -> `dir`, `pwd` -> `cd`.
- Команды вида `tools/.../program.exe` автоматически преобразуются в абсолютный путь от корня приложения, чтобы они не искались внутри папки проекта.
- Вкладка «Web»: добавлен переключатель «Исходный код / Страница».
- Web-страница теперь показывается в упрощённом браузерном виде: HTML очищается от `script/style`, теги преобразуются в читаемый текст; исходный HTML доступен отдельной кнопкой.
- Потоковые ответы больше не пишут весь `reasoning_content` в лог. Логируются только счётчики reasoning/content/tool chunks и небольшой образец content/tool delta.
- Изменения файлов теперь группируются в одном сообщении прогресса и прикрепляются к итоговому скрытому блоку, вместо большого количества сообщений «Изменён файл: ...».
- При запуске приложения полный scan `tools` отложен до момента, когда tools действительно нужны; окно должно открываться быстрее.
- При переключении проекта старый проект выгружается, новый показывается как загружаемый через `projectLoading` и `LinearProgressIndicator`.
- Добавлен кэш сканирования `tools` на короткое время, чтобы не сканировать MSYS2 повторно на каждом действии.

Ограничение: полноценный нативный WebView как Chromium/Edge требует отдельного Flutter-плагина/платформенной реализации. В этой версии сделан встроенный browser-like HTML/text viewer без внешнего плагина и с переключением на исходный код.
