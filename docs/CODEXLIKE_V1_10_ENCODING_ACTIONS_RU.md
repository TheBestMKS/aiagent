# v1.10 — кодировка консоли Windows и сворачиваемые действия агента

## Исправление вывода команд Windows

В Windows `cmd.exe` часто пишет русский текст в OEM-кодировке CP866. Раньше вывод декодировался через `systemEncoding`, из-за чего сообщение:

```text
"g++" не является внутренней или внешней командой, исполняемой программой или пакетным файлом.
```

могло превращаться в кракозябры вида:

```text
"g++" ­Ґ пў«пҐвбп ў­гваҐ­­Ґ© ...
```

Из-за этого модель воспринимала ошибку окружения как ошибку кода и предлагала неверные действия.

Добавлено:

- декодирование CP866 для вывода `cmd.exe`;
- выбор лучшей декодировки по эвристике качества;
- нормальная передача stdout/stderr модели;
- корректное распознавание ошибок окружения: отсутствует `g++`, `cl.exe`, `clang++`, `cmake`, `flutter`, `npm` и т.п.

## Сворачиваемые действия в диалоге

Добавлены Codex-like блоки действий:

- команда запуска;
- запись файла;
- чтение файла;
- автопроверка проекта;
- удаление/копирование/перемещение;
- создание папки.

Если одно действие выполняется повторно, оно отображается одним блоком с количеством попыток. Внутри раскрываемого блока отображаются результаты каждой попытки, включая stdout/stderr и exit code.

Блок раскрывается и скрывается нажатием по действию.

## Форматирование текста

Видимые сообщения теперь поддерживают базовое форматирование:

- `**жирный текст**`;
- `` `моноширинный текст` ``.

Это позволяет нормально отображать сообщения вида:

```text
**Причина неудачи:** компилятор g++ не найден в PATH.
```

## Контекст модели

Для LM Studio `/api/v0/models` агент теперь учитывает `loaded_context_length` конкретной загруженной модели. Если модель по паспорту поддерживает 131072 токена, но сервер загрузил её с маленьким runtime-контекстом, агент честно сообщает об этом и не пытается сжать Codex/tools-режим до нескольких сотен токенов.
